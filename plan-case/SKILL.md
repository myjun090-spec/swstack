# /plan-case — 서비스 개입계획 수립 스킬

> 역할: 사례관리 슈퍼바이저
> 단계: 사례관리 파이프라인 3단계 (접수 → 사정 → **계획** → 개입)

## 개요

사정보고서(`assessment.md`)를 기반으로 클라이언트와 함께 개입계획을 수립합니다.
목표-하위목표-활동 구조를 체계적으로 설계하고, 서비스 연계 계획을 포함합니다.

## 실행 전 확인

1. `.swstack/cases/{case-id}/assessment.md`를 읽습니다
2. 사정보고서가 없으면 `/assessment`를 먼저 실행하도록 안내합니다
3. 사정보고서의 욕구 우선순위와 강점 요약을 핵심 입력으로 사용합니다

## 워크플로

### Step 1: 사정 결과 요약 리뷰

assessment.md에서 다음을 추출합니다:
- 욕구 우선순위 매트릭스 (클라이언트 동의 항목 중심)
- 강점/자원 요약
- 위험요인
- 사례 유형 (집중/일반)
- 권장 서비스

### Step 2: 목표 수립

**SMART 원칙**에 따라 목표를 수립합니다:
- **S**pecific (구체적): 명확하게 측정 가능한 행동/상태
- **M**easurable (측정 가능): 달성 여부를 판단할 수 있는 기준
- **A**chievable (달성 가능): 현실적으로 도달 가능한 수준
- **R**elevant (관련성): 클라이언트의 욕구와 직접 연결
- **T**ime-bound (기한): 달성 기한 명시

목표 구조:
```
장기목표 (6개월~1년)
├── 단기목표 1 (1~3개월)
│   ├── 활동 1-1
│   ├── 활동 1-2
│   └── 활동 1-3
├── 단기목표 2 (1~3개월)
│   ├── 활동 2-1
│   └── 활동 2-2
└── 단기목표 3 (1~3개월)
    └── 활동 3-1
```

사용자에게 각 목표에 대해 질문합니다:

1. "클라이언트가 가장 먼저 해결하고 싶은 것은 무엇인가요?"
2. "6개월 후 어떤 상태가 되면 성공이라고 할 수 있을까요?"
3. "지금 바로 시작할 수 있는 작은 단계는 무엇인가요?"

### Step 3: 서비스 연계 계획

각 활동에 대해 다음을 결정합니다:

| 활동 | 서비스 유형 | 제공 주체 | 빈도 | 기간 |
|------|-----------|----------|------|------|
| | 직접서비스/연계/의뢰 | 기관명/담당자 | 주1회 등 | 시작~종료 |

서비스 유형:
- **직접서비스**: 우리 기관이 직접 제공 (상담, 프로그램 등)
- **연계**: 타 기관 서비스를 연결하고 모니터링
- **의뢰**: 타 기관으로 이관 (우리 기관 역할 종료)

### Step 4: 역할 분담

| 역할 | 담당 | 구체적 책임 |
|------|------|------------|
| 클라이언트 | {이니셜} | (자기 역할 — 자기결정 원칙) |
| 담당 사회복지사 | {이름} | |
| 슈퍼바이저 | {이름} | |
| 연계기관 담당자 | {기관/이름} | |
| 가족/지지체계 | {관계} | |

### Step 5: 모니터링 계획

| 모니터링 항목 | 방법 | 주기 | 기준 |
|-------------|------|------|------|
| 목표 달성도 | 면담/전화 | 월 1회 | 목표달성척도 |
| 서비스 이용 현황 | 출석/기록 | 매회기 | 참여율 |
| 위기 징후 | 면담 | 매회기 | 위험요인 체크 |
| 만족도 | 설문 | 분기 1회 | 만족도 척도 |

### Step 6: 계획 동의 확인

클라이언트와 함께 계획을 검토했는지 확인합니다:

- [ ] 클라이언트에게 계획 내용을 설명함
- [ ] 클라이언트가 목표에 동의함
- [ ] 클라이언트가 자신의 역할을 이해함
- [ ] 계획 수정 요청 사항 반영 완료
- [ ] 서비스 이용 동의서 징구 완료

## 계획 수정

기존 계획을 수정할 때는:
1. 이전 plan.md를 `.swstack/cases/{case-id}/plans/plan-{날짜}.md`로 아카이브합니다
2. 수정 사유를 기록합니다
3. `plan_type: 수정계획`으로 표시합니다
4. 변경된 목표/활동을 하이라이트합니다

## 슈퍼비전 체크포인트

계획 수립 완료 후, 슈퍼바이저 검토 질문을 제시합니다:

1. 목표가 클라이언트의 욕구를 정확히 반영하는가?
2. 클라이언트의 강점이 계획에 활용되고 있는가?
3. 목표가 현실적이고 측정 가능한가?
4. 위기 대응 계획이 충분한가?
5. 연계 자원이 적절한가?

## 연결 스킬

- **이전 단계**: `/assessment` — 사정보고서 참조
- **다음 단계**: `/intervene` — 개입 기록 시작
- **참고**: `/resource-map` — 연계 자원 조회
- **참고**: `/case-conference` — 복잡 사례 시 사례회의 요청
- **점검**: `/evaluate` — 목표달성도 중간평가
