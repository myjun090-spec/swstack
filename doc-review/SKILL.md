# /doc-review — 문서 다단계 검토 스킬

> 역할: 문서 검토 전문가 (7인 패널)
> 유형: 지식그래프 (다단계 품질 검토)

## 개요

작성된 문서를 7단계 검토 체크리스트로 다각도 심사합니다.
각 검토 단계는 독립적인 전문가 관점에서 수행되며, 영역별 점수와 구체적 개선 제안을 제공합니다.
I'm Author의 7단계 검토 프레임워크에서 영감을 받아 사회복지 문서에 특화했습니다.

## 실행 조건

- 검토할 문서가 존재해야 합니다 (파일 경로 또는 텍스트 직접 입력)
- 모든 유형의 문서에 적용 가능합니다: 보고서, 계획서, 기록물, 홍보물 등

## 워크플로

### Step 1: 문서 접수

사용자에게 다음을 질문합니다:

1. **검토 대상 문서**: 파일 경로 또는 텍스트 직접 입력
2. **문서 유형**: 접수기록 / 사정보고서 / 개입계획서 / 평가보고서 / 정부보고서 / 사업계획서 / 홍보물 / 기타
3. **검토 목적**: 내부 검토 / 외부 제출 전 점검 / 슈퍼비전 / 품질 개선
4. **특별히 중점 검토할 영역**: (선택사항)

### Step 2: 7단계 검토 실행

각 단계를 순서대로 실행합니다. 각 단계는 독립적 전문가 관점으로 검토합니다.

#### 검토 1: 정확성 (Accuracy) 검토 — 사실 확인 전문가

- [ ] 날짜, 숫자, 통계가 정확한가
- [ ] 사례 정보가 다른 기록(intake, assessment 등)과 일치하는가
- [ ] 인용된 법률, 제도, 기준이 현행 법령에 부합하는가
- [ ] 기관명, 프로그램명, 담당자명이 정확한가
- [ ] 척도 점수와 해석이 일치하는가 (예: PHQ-9 점수와 우울 수준 분류)

**검토 기준**:
| 점수 | 기준 |
|------|------|
| 5 | 모든 사실이 정확하고 검증 가능 |
| 4 | 사소한 오류 1-2건 (수정 용이) |
| 3 | 확인이 필요한 정보가 있음 |
| 2 | 주요 사실 관계 오류가 있음 |
| 1 | 심각한 사실 오류가 다수 |

#### 검토 2: 근거 (Evidence) 검토 — 근거기반실천 전문가

- [ ] 주장과 판단에 근거가 명시되어 있는가
- [ ] 사정 결과가 관찰과 데이터에 기반하는가
- [ ] 개입 방법 선택에 이론적/실증적 근거가 있는가
- [ ] 평가 결과가 표준화 척도에 기반하는가
- [ ] "느낌" 대신 구체적 관찰과 데이터를 사용하는가

**검토 기준**:
| 점수 | 기준 |
|------|------|
| 5 | 모든 판단에 명확한 근거 제시, 표준화 척도 적절히 활용 |
| 4 | 대부분 근거 있으나, 일부 보완 필요 |
| 3 | 근거가 부족한 판단이 있음 |
| 2 | 주관적 판단이 다수, 근거 부족 |
| 1 | 근거 없는 주장이 대부분 |

#### 검토 3: 구조 (Structure) 검토 — 문서구조 전문가

- [ ] 논리적 흐름이 자연스러운가 (서론→본론→결론)
- [ ] 목차/섹션이 체계적으로 구성되어 있는가
- [ ] 불필요한 중복이 없는가
- [ ] 핵심 내용이 빠짐없이 포함되어 있는가
- [ ] 표, 그래프, 목록이 적절히 활용되어 있는가

**검토 기준**:
| 점수 | 기준 |
|------|------|
| 5 | 체계적 구조, 논리적 흐름, 빈틈 없음 |
| 4 | 전반적으로 좋으나 소소한 구조 개선 가능 |
| 3 | 구조는 있으나 논리적 흐름에 개선 필요 |
| 2 | 구조가 불명확하거나 주요 섹션 누락 |
| 1 | 구조화되지 않은 문서 |

#### 검토 4: 문법/표현 (Grammar) 검토 — 교정 전문가

- [ ] 맞춤법, 띄어쓰기 오류는 없는가
- [ ] 문장이 명확하고 간결한가 (불필요한 중복 표현 제거)
- [ ] 능동태 위주로 작성되었는가
- [ ] 전문용어 사용이 적절한가 (과도하거나 부족하지 않은지)
- [ ] 존댓말/반말이 일관적인가

**검토 기준**:
| 점수 | 기준 |
|------|------|
| 5 | 오류 없음, 명확한 표현, 가독성 우수 |
| 4 | 사소한 오류 1-3건 |
| 3 | 수정이 필요한 표현이 여러 곳 |
| 2 | 가독성에 영향을 주는 오류 다수 |
| 1 | 전반적인 문장 재작성 필요 |

#### 검토 5: 일관성 (Consistency) 검토 — 품질관리 전문가

- [ ] 용어 사용이 문서 전체에서 일관적인가 (같은 개념에 다른 용어 사용 여부)
- [ ] 형식(번호 매기기, 날짜 형식, 표 구조)이 통일되어 있는가
- [ ] 이전 단계 산출물(접수→사정→계획)과 내용이 일관적인가
- [ ] 목표와 평가 기준이 연결되어 있는가
- [ ] 숫자와 통계가 문서 내에서 일치하는가

**검토 기준**:
| 점수 | 기준 |
|------|------|
| 5 | 완벽한 일관성, 이전 문서와도 정합성 유지 |
| 4 | 사소한 불일치 1-2건 |
| 3 | 용어나 형식 불일치가 여러 곳 |
| 2 | 주요 내용의 불일치가 있음 |
| 1 | 전반적으로 일관성 부족 |

#### 검토 6: 전문성 (Professionalism) 검토 — 사회복지 전문가

- [ ] 사회복지 윤리강령에 부합하는가
- [ ] 강점관점이 반영되어 있는가 (문제 중심이 아닌 강점 중심)
- [ ] 낙인표현이 없는가 ("문제 가정", "결손 가정" 등)
- [ ] 클라이언트의 자기결정권이 존중되는 서술인가
- [ ] 문화적 역량이 반영되어 있는가 (다양성 존중)
- [ ] 인간 존엄성에 기반한 표현을 사용하는가

**검토 기준**:
| 점수 | 기준 |
|------|------|
| 5 | 전문적 기준 완벽 충족, 강점관점 잘 반영 |
| 4 | 전반적으로 전문적이나 일부 표현 개선 가능 |
| 3 | 전문적 기준 일부 미충족 |
| 2 | 낙인표현 또는 윤리적 문제가 있음 |
| 1 | 전문적 기준에 심각하게 미달 |

#### 검토 7: 법적 적합성 (Legal Compliance) 검토 — 법률 검토 전문가

- [ ] 개인정보보호법 준수 여부 (실명, 주민번호, 주소 등 민감정보)
- [ ] 사회복지사업법 제47조(비밀누설 금지) 준수 여부
- [ ] 의무신고 관련 법률 준수 여부 (아동학대, 노인학대 등)
- [ ] 동의서 관련 사항이 적절히 기록되어 있는가
- [ ] 보조금 관리에 관한 법률 준수 여부 (해당 시)
- [ ] 문서 보관 기한 관련 법적 요건 충족 여부

**검토 기준**:
| 점수 | 기준 |
|------|------|
| 5 | 모든 법적 요건 충족, 개인정보 완벽 보호 |
| 4 | 법적 요건 충족, 사소한 보완 사항 |
| 3 | 확인이 필요한 법적 사항이 있음 |
| 2 | 법적 요건 미충족 사항이 있음 (수정 필요) |
| 1 | 심각한 법적 위반 가능성 (즉시 수정 필요) |

### Step 3: 종합 평가

7개 영역의 점수를 종합하여 전체 평가를 산출합니다.

### Step 4: 수정 제안

각 영역에서 발견된 문제에 대해 구체적인 수정 제안을 제공합니다.

## 연결 스킬

- **대상**: 모든 스킬의 산출물을 검토할 수 있습니다
- **데이터**: `/knowledge-index` — 이전 기록과의 일관성 확인 시 참조
- **보호**: `/privacy-guard` — 법적 적합성 검토 시 연계
- **참고**: `/law-check` — 법적 적합성 검토의 상세 법률 확인
- **활용**: `/auto-report` — 자동 생성 보고서의 품질 검증에 활용
