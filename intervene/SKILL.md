# /intervene — 개입 기록 스킬

> 역할: 실천가
> 단계: 사례관리 파이프라인 4단계 (계획 → **개입** → 평가)

## 개요

개입계획서(`plan.md`)에 따른 서비스 제공 과정을 회기별로 기록합니다.
프로세스 노트(과정기록) 작성, 진행상황 추적, 변화 측정을 수행합니다.

## 실행 전 확인

1. `.swstack/cases/{case-id}/plan.md`를 읽습니다
2. 이전 개입 기록(notes/)이 있으면 최근 기록을 참조합니다
3. 계획서의 목표와 활동을 확인합니다

## 워크플로

### Step 1: 회기 정보 입력

사용자에게 다음을 질문합니다:

1. 회기 번호 (자동 카운트 제안)
2. 일시
3. 상담 형태: 대면 / 전화 / 방문 / 화상 / 기타
4. 소요 시간

### Step 2: 개입 내용 기록

| 항목 | 설명 |
|------|------|
| 클라이언트 상태 | 면담 시 관찰된 정서, 태도, 외형 변화 |
| 주요 논의 내용 | 이번 회기에서 다룬 주제 |
| 제공 서비스 | 계획서의 어떤 활동을 실행했는지 |
| 클라이언트 반응 | 서비스에 대한 반응, 참여도 |
| 변화 관찰 | 지난 회기 대비 변화 (개선/유지/악화) |
| 과제/약속 | 다음 회기까지 클라이언트/실천가 과제 |

### Step 3: 목표 진행률 업데이트

plan.md의 각 목표에 대해 진행 상황을 기록합니다:

```
목표 1: {목표 내용}
진행률: ████████░░ 80%
상태: 순조 / 지연 / 장벽발생 / 달성
메모: {구체적 진행 내용}
```

### Step 4: 위험요인 재점검

매 회기마다 간략하게 확인합니다:
- 새로운 위기 징후가 있는가?
- 기존 위험요인의 변화가 있는가?

변화가 감지되면 즉시 AskUserQuestion으로 알립니다.

### Step 5: 다음 회기 계획

- 다음 회기 예정일
- 다음 회기에서 다룰 내용
- 준비 사항

## 산출물 형식

`.swstack/cases/{case-id}/notes/{YYYY-MM-DD}-session-{N}.md`에 저장합니다.

```markdown
---
case_id: {case-id}
session_number: {N}
session_date: {날짜}
session_type: 대면/전화/방문/화상
duration: {분}
worker: {담당자}
---

# 상담기록 — 제{N}회기

## 1. 클라이언트 상태 관찰
{면담 시 관찰 내용}

## 2. 주요 논의 내용
{이번 회기 주제 및 내용}

## 3. 제공 서비스
- 계획 활동: {plan.md의 해당 활동}
- 실제 수행: {구체적 수행 내용}

## 4. 클라이언트 반응
{서비스에 대한 반응, 참여도}

## 5. 변화 관찰
{지난 회기 대비 변화}

## 6. 목표 진행 현황
| 목표 | 진행률 | 상태 |
|------|--------|------|
| {목표1} | {%} | 순조/지연/달성 |
| {목표2} | {%} | 순조/지연/달성 |

## 7. 위험요인 점검
- 새로운 위기 징후: 없음 / {있으면 기술}
- 기존 위험요인 변화: 없음 / {변화 기술}

## 8. 다음 회기 계획
- 예정일: {날짜}
- 다룰 내용: {계획}
- 과제: {클라이언트/실천가 과제}

## 9. 실천가 소견
{전문적 판단, 개입 방향 조정 필요성 등}
```

## 특수 기록 유형

### 위기개입 기록
위기 상황 발생 시 별도 양식:
- 위기 상황 설명
- 즉각 개입 내용
- 안전 확인 결과
- 후속 계획

### 서비스 연계 기록
타 기관 연계 시:
- 연계 기관 및 서비스
- 연계 사유
- 연계 결과 (수락/대기/거절)
- 모니터링 계획

## 연결 스킬

- **이전 단계**: `/plan-case` — 개입계획 참조
- **병행**: `/case-conference` — 어려운 사례 시 사례회의 요청
- **다음 단계**: `/evaluate` — 충분한 회기 후 중간/최종 평가
- **위기 시**: 위기개입 프로토콜 즉시 실행
