# /auto-report — 자동 보고서 생성 스킬

> 역할: 보고서 작성 전문가
> 유형: 지식그래프 (자동 문서 생성)

## 개요

축적된 사례 지식과 기관 데이터를 기반으로 정형화된 보고서를 자동 생성합니다.
사용자가 최소한의 입력(기간, 대상, 특이사항)만 제공하면, 지식 베이스에서 필요한 데이터를 수집하여 보고서 초안을 완성합니다.

## 지원 보고서 템플릿

| 번호 | 템플릿 | 설명 | 주요 데이터 소스 |
|------|--------|------|----------------|
| 1 | 주간업무보고서 | 주간 사례관리/행정 업무 요약 | 상담기록, 회의록, 일정 |
| 2 | 사례회의록 | 사례회의 결과 정리 | 사례 기록, 회의 논의 내용 |
| 3 | 정부보고서 | 행복e음 실적, 분기/연간 보고 | 사례 통계, 서비스 실적 |
| 4 | 사업계획서 | 차년도/신규 사업계획 | 성과 데이터, 욕구조사, 지역분석 |
| 5 | 결산보고서 | 사업별 예산 집행 결과 | 예산, 지출, 성과 데이터 |

## 실행 조건

- `.swstack/knowledge/catalog.json`이 있으면 지식 베이스를 활용합니다
- 지식 베이스가 없어도 사용자 직접 입력으로 작성 가능합니다

## 워크플로

### Step 1: 템플릿 선택

사용자에게 보고서 유형을 질문합니다:

```
어떤 보고서를 작성하시겠습니까?
1) 주간업무보고서
2) 사례회의록
3) 정부보고서 (분기/연간)
4) 사업계획서
5) 결산보고서
```

### Step 2: 기본 정보 수집

템플릿별로 필요한 최소 정보를 수집합니다:

#### 주간업무보고서
- 보고 기간 (예: 2026.03.25 ~ 03.29)
- 작성자/부서
- 특이사항이나 강조할 내용

#### 사례회의록
- 회의 일시, 참석자
- 상정 사례 목록 (case-id 또는 간략 설명)
- 주요 논의 사항

#### 정부보고서
- 보고 기간 (분기/연간)
- 보고 대상 사업
- 특별 성과 또는 이슈

#### 사업계획서
- 사업명, 사업 기간
- 예산 규모
- 핵심 목표

#### 결산보고서
- 결산 기간
- 대상 사업
- 특이 집행 사항

### Step 3: 지식 베이스에서 데이터 수집

카탈로그와 색인 파일에서 관련 데이터를 자동 수집합니다:

| 보고서 유형 | 수집 데이터 |
|-----------|-----------|
| 주간업무보고서 | 해당 주간 상담기록, 회의록, 연계 현황, 사례 상태 변화 |
| 사례회의록 | 해당 사례의 접수-사정-계획-개입-평가 기록 전체 |
| 정부보고서 | 사례 유형별 통계, 서비스 실적, 목표달성률, 만족도 |
| 사업계획서 | 전년도 성과 데이터, 욕구 분포, 지역 자원 현황 |
| 결산보고서 | 예산 편성 대비 집행 현황, 사업별 성과 지표 |

### Step 4: 보고서 초안 생성

템플릿에 데이터를 채워 초안을 생성합니다. 아래는 각 템플릿의 상세 구조입니다.

### Step 5: 검토 및 수정

생성된 초안을 사용자에게 보여주고 수정 요청을 받습니다:
- "내용을 추가/삭제/수정해주세요"
- "이 부분을 더 구체적으로 작성해줘"
- "어조를 더 공식적으로 바꿔줘"

## 사용 예시

```
사용자: /auto-report
AI: 어떤 보고서를 작성하시겠습니까?
    1) 주간업무보고서  2) 사례회의록  3) 정부보고서
    4) 사업계획서      5) 결산보고서

사용자: 주간업무보고서, 이번 주 (3/25~3/29)

AI: 이번 주(3/25~3/29) 주간업무보고서를 작성합니다.
    작성자와 부서를 알려주세요.

사용자: 김복지, 사례관리팀

AI: 지식 베이스에서 이번 주 데이터를 수집합니다...
    ├── 신규 접수: 2건
    ├── 상담 기록: 8건 (대면 5, 전화 2, 방문 1)
    ├── 사례회의: 1건 (3/27)
    ├── 종결: 1건
    └── 서비스 연계: 3건

    [주간업무보고서 초안이 생성되었습니다]
    수정이 필요한 부분이 있으시면 말씀해주세요.
    특이사항이나 추가할 내용이 있으시면 알려주세요.
```

## 개인정보 보호 규칙

- 보고서에 클라이언트 실명은 포함하지 않습니다 (case_id 사용)
- 정부보고서의 통계는 비식별 집계 데이터만 사용합니다
- 사례회의록에서 민감 정보는 자동 마스킹합니다
- 외부 제출용 보고서 생성 시 `/privacy-guard`를 자동 실행합니다

## 연결 스킬

- **데이터 소스**: `/knowledge-index` — 색인된 지식 베이스
- **데이터 소스**: `/knowledge-query` — 필요 데이터 검색
- **검토**: `/doc-review` — 생성된 보고서 품질 검토
- **참고**: `/budget`, `/expense` — 회계 데이터 참조
- **참고**: `/report-gov` — 정부보고서 양식 참고
- **참고**: `/proposal` — 사업계획서 양식 참고
