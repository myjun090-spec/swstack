# /intake — 접수상담 스킬

> 역할: 접수상담원
> 단계: 사례관리 파이프라인의 첫 번째 단계

## 개요

클라이언트의 초기 접수상담을 구조화된 형식으로 기록합니다.
접수 후 산출물은 `/assessment` 스킬의 입력으로 자동 연결됩니다.

## 실행 조건

- 이 스킬은 새로운 사례를 시작할 때 사용합니다
- 기존 사례에 대한 추가 접수는 case-id를 지정하여 실행합니다

## 워크플로

### Step 1: 기본 정보 수집

사용자에게 다음 정보를 순서대로 질문합니다. 모르는 항목은 "미확인"으로 기록합니다.

1. **의뢰 경로**: 자발적 내방 / 기관 의뢰 / 주민센터 연계 / 1577-0199 / 기타
2. **클라이언트 기본 정보**:
   - 성별, 연령대 (실명/주민번호는 기록하지 않음)
   - 가구 구성 (예: 1인가구, 조손가구, 한부모가구 등)
   - 거주 지역 (구/동 단위까지만)
3. **주 호소 문제**: 클라이언트가 직접 표현한 어려움 (원문 그대로 기록)
4. **희망 서비스**: 클라이언트가 원하는 지원

### Step 2: 긴급도 분류

다음 기준으로 긴급도를 판정합니다:

| 등급 | 기준 | 대응 |
|------|------|------|
| 🔴 위기 | 자살위험, 학대/방임, 노숙, 급성 정신건강 위기 | 즉시 개입, 긴급 자원 연결 |
| 🟡 긴급 | 경제적 위기, 건강 악화, 주거 불안정 | 48시간 내 사정 실시 |
| 🟢 일반 | 정보 제공, 프로그램 참여, 예방적 서비스 | 1주 내 사정 실시 |

**위기 등급 감지 시**: 즉시 AskUserQuestion으로 실천가에게 알립니다.
```
⚠️ 위기 상황이 감지되었습니다.
감지 내용: [구체적 내용]
즉시 위기개입 프로토콜을 실행하시겠습니까?
관련 긴급연락처: 자살예방상담전화 1393, 아동학대 112, 여성긴급전화 1366
```

### Step 3: 동의서 체크리스트

다음 동의 항목을 확인합니다:

- [ ] 개인정보 수집·이용 동의
- [ ] 개인정보 제3자 제공 동의
- [ ] 사례관리 서비스 이용 동의
- [ ] 사진/영상 촬영 동의 (해당 시)

### Step 4: 접수기록서 생성

수집된 정보를 아래 형식으로 `.swstack/cases/{case-id}/intake.md`에 저장합니다.

## 산출물 형식

```markdown
---
case_id: {자동생성 YYYYMMDD-NNN}
intake_date: {날짜}
worker: {담당자명}
urgency: {위기/긴급/일반}
status: 접수완료
---

## 1. 의뢰 경로
{의뢰 경로 및 경위}

## 2. 클라이언트 기본 정보
- 성별/연령대:
- 가구 구성:
- 거주 지역:
- 주요 특성: (장애, 다문화, 독거 등 해당 시)

## 3. 주 호소 문제
> "{클라이언트 원문 표현}"

{실천가 관점의 문제 요약}

## 4. 희망 서비스
{클라이언트가 표현한 필요 서비스}

- 등급: {위기/긴급/일반}
- 판정 근거: {구체적 근거}

- [x/] 사례관리 서비스 이용 동의

```

## 개인정보 보호 규칙

- 클라이언트 실명 대신 case_id를 사용합니다
- 주민등록번호, 상세 주소, 연락처는 기록하지 않습니다
- 이 파일에 기록된 정보만으로는 개인을 특정할 수 없어야 합니다
- 실명이 필요한 경우 별도 보안 시스템에 기록하도록 안내합니다

## 연결 스킬

- **다음 단계**: `/assessment` — 접수기록서를 기반으로 욕구사정 실시
- **위기 시**: 즉시 위기개입 프로토콜 안내
- **참고**: `/resource-map` — 의뢰 가능한 지역사회 자원 조회
