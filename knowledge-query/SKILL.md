# /knowledge-query — 지식 검색 스킬

> 역할: 지식 검색 전문가
> 유형: 지식그래프 (Knowledge Graph + RAG)

## 개요

축적된 사례 지식 베이스에서 정보를 검색하고, 유사 사례를 찾고, 근거 기반 답변을 생성합니다.
`/knowledge-index`로 색인된 데이터와 원본 문서를 함께 참조하여 RAG(검색증강생성) 방식으로 응답합니다.
사회복지사가 "비슷한 사례 있었나?", "이런 상황에서 어떤 개입이 효과적이었지?" 같은 질문에 답합니다.

## 실행 조건

- `.swstack/knowledge/catalog.json`이 존재해야 합니다
- 색인된 사례가 없으면 `/knowledge-index`를 먼저 실행하도록 안내합니다

## 워크플로

### Step 1: 질의 접수

사용자의 질문을 분석하여 검색 유형을 판별합니다:

| 검색 유형 | 판별 기준 | 예시 질문 |
|----------|----------|----------|
| 유사 사례 검색 | "비슷한", "유사한", "같은 사례" | "독거노인 우울 사례 중 비슷한 거 찾아줘" |
| 개입 방법 검색 | "개입", "방법", "효과적", "어떻게" | "알코올 의존 클라이언트에게 효과적인 개입은?" |
| 성과 분석 | "성과", "효과", "결과", "달성률" | "심리상담의 평균 목표달성률은?" |
| 자원 검색 | "자원", "기관", "연계", "어디" | "긴급생계지원 연계한 기관 목록 보여줘" |
| 통계 질의 | "몇 건", "평균", "많이", "분포" | "올해 위기사례가 몇 건이었어?" |
| 자유 질의 | 위에 해당하지 않는 일반 질문 | "지난 분기 주요 이슈가 뭐였지?" |

### Step 2: 검색 실행

#### 2-1. 카탈로그 검색
`.swstack/knowledge/catalog.json`에서 태그, 욕구 영역, 서비스 유형 기반으로 관련 사례를 필터링합니다.

#### 2-2. 색인 파일 검색
필터링된 사례의 `.swstack/knowledge/index/{case-id}-index.json`을 읽어 엔티티와 관계를 분석합니다.

#### 2-3. 원본 문서 참조
관련성이 높은 사례의 원본 문서(intake.md, assessment.md, plan.md 등)를 읽어 상세 컨텍스트를 확보합니다.

### Step 3: 답변 생성

검색 결과를 종합하여 다음 구조로 답변합니다:

1. **직접 답변**: 질문에 대한 명확한 답변
2. **근거 사례**: 답변의 근거가 된 사례 목록 (case_id, 핵심 정보)
3. **패턴 분석**: 여러 사례에서 발견된 공통 패턴
4. **실천 제안**: 근거에 기반한 구체적 실천 제안
5. **주의사항**: 일반화의 한계, 개별화 필요성 안내

### Step 4: 결과 출력

## 산출물 형식

검색 결과는 대화로 직접 전달하며, 필요 시 `.swstack/knowledge/queries/` 디렉토리에 저장합니다.

```markdown
---
query_date: {날짜}
query_type: {검색 유형}
query_text: "{원본 질문}"
results_count: {검색된 사례 수}
---

# 지식 검색 결과

## 질문
> "{사용자 질문}"

## 답변

### 핵심 답변
{질문에 대한 직접적인 답변}

### 관련 사례

| 사례 | 핵심 특성 | 개입 방법 | 성과 | 관련도 |
|------|----------|----------|------|--------|
| CASE-20260301-001 | 독거노인, 우울 | 인지행동치료 주1회 | PHQ-9: 18→8 | 높음 |
| CASE-20251105-012 | 독거노인, 우울+경제 | 상담+긴급지원 | PHQ-9: 15→7 | 높음 |
| CASE-20250820-008 | 노인, 우울+고립 | 집단프로그램 | PHQ-9: 12→6 | 중간 |

### 패턴 분석
{여러 사례에서 발견된 공통적인 성공/실패 요인}

- **효과적이었던 요소**: {공통 성공 요인}
- **어려웠던 점**: {공통 장벽}
- **핵심 변수**: {성과에 영향을 미친 주요 변수}

### 실천 제안
{근거에 기반한 구체적 제안}

1. {제안 1} — 근거: {관련 사례/데이터}
2. {제안 2} — 근거: {관련 사례/데이터}
3. {제안 3} — 근거: {관련 사례/데이터}

### 주의사항
- 위 분석은 {N}건의 사례를 기반으로 한 것이며, 개별 클라이언트의 고유한 상황을 반드시 고려해야 합니다
- 통계적 분석이 아닌 사례 기반 참고자료입니다
- 최종 판단은 담당 사회복지사의 전문적 판단에 따릅니다
```

## 자주 쓰는 질의 예시

```
사용자: /knowledge-query
AI: 어떤 정보를 찾으시겠습니까?

사용자: 독거노인 우울 사례에서 효과적이었던 개입 방법을 찾아줘

AI: ## 검색 결과: 독거노인 우울 사례 개입 방법

  ### 핵심 답변
  색인된 45건 중 독거노인 우울 관련 사례 7건을 분석한 결과,
  가장 효과적이었던 개입 방법은 다음과 같습니다:

  1. **인지행동치료(CBT) 기반 개인상담** (5건 적용, 평균 PHQ-9 감소: 9.2점)
  2. **사회적 지지체계 구축** (4건 적용, 사회적 고립도 평균 개선)
  3. **복합 서비스 연계** (심리+경제+돌봄, 3건, 종결률 가장 높음)

  ### 관련 사례
  | 사례 | 개입 방법 | PHQ-9 변화 | 성과 |
  |------|----------|-----------|------|
  | CASE-20260301-001 | CBT 주1회 + 돌봄서비스 | 18→8 | 종결 |
  | CASE-20251105-012 | CBT + 긴급생계지원 | 15→7 | 종결 |
  | CASE-20250820-008 | 집단프로그램 + 자조모임 | 12→6 | 종결 |
  ...

  ### 실천 제안
  1. 인지행동치료를 주 1회 이상 실시 — 5건 모두에서 유의미한 효과
  2. 사회적 고립 해소를 병행 — 상담 단독보다 복합 개입이 효과적
  3. 경제적 지원을 초기에 병행 — 기본 욕구 해소가 상담 효과에 영향

  ⚠️ 주의: 7건의 사례 기반이므로 일반화에 한계가 있습니다.
  클라이언트의 개별 상황에 맞춘 판단이 필요합니다.
```

## 개인정보 보호 규칙

- 검색 결과에 클라이언트 실명, 주민번호, 상세 주소는 포함하지 않습니다
- case_id로만 사례를 식별하며, 비식별 특성만 표시합니다
- 검색 로그에는 질의 내용과 결과 요약만 저장합니다

## 연결 스킬

- **전제**: `/knowledge-index` — 검색 대상 지식 색인
- **활용**: `/auto-report` — 검색 결과를 보고서에 반영
- **참고**: `/resource-map` — 자원 관련 질의 시 보완
- **참고**: `/assessment` — 유사 사례의 사정 정보 참고
- **참고**: `/plan-case` — 유사 사례의 개입계획 참고
