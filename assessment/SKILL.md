# /assessment — 욕구사정 스킬

> 역할: 사례관리자
> 단계: 사례관리 파이프라인 2단계 (접수 → **사정** → 계획)

## 개요

접수기록서(`intake.md`)를 기반으로 클라이언트의 욕구, 강점, 자원, 위험요인을 종합적으로 사정합니다.
강점관점과 생태체계 관점을 통합하여, 개입계획 수립의 근거를 마련합니다.

## 실행 전 확인

1. `.swstack/cases/{case-id}/intake.md` 파일을 읽습니다
2. 접수기록서가 없으면 `/intake`를 먼저 실행하도록 안내합니다
3. 접수기록서의 긴급도가 "위기"인 경우, 위기개입이 완료되었는지 확인합니다

## 워크플로

### Step 1: 접수기록 리뷰

intake.md를 읽고 다음을 파악합니다:
- 주 호소 문제
- 클라이언트 기본 정보
- 긴급도 등급
- 초기 소견

### Step 2: 심층 사정 인터뷰

사용자(실천가)에게 다음 영역별로 질문합니다.
각 영역에서 **문제/욕구**와 **강점/자원**을 동시에 파악합니다.

#### 2-1. 신체적 영역
- 건강 상태, 질병/장애 여부
- 일상생활 수행 능력 (ADL/IADL)
- 의료 접근성

#### 2-2. 심리·정서적 영역
- 정서 상태 (우울, 불안, 스트레스)
- 자아존중감, 자기효능감
- 트라우마 이력
- 표준화 척도 활용:
  - PHQ-9 (우울): 0-4 최소 / 5-9 경미 / 10-14 중등도 / 15-19 중등도-심함 / 20-27 심함
  - GAD-7 (불안): 0-4 최소 / 5-9 경미 / 10-14 중등도 / 15-21 심함
  - 필요시 추가 척도 제안

#### 2-3. 경제적 영역
- 소득원 및 수준 (수급/차상위/일반)
- 부채, 경제적 위기 요인
- 경제적 자립 가능성

#### 2-4. 사회·관계적 영역
- 가족 관계 및 기능
- 사회적 지지체계 (친구, 이웃, 종교 등)
- 사회적 고립 정도
- 갈등 관계

#### 2-5. 주거·환경적 영역
- 주거 형태 및 안정성
- 주거 환경 (안전, 위생, 접근성)
- 지역사회 인프라 접근성

#### 2-6. 직업·교육적 영역
- 취업 상태, 직업 이력
- 교육 수준, 직업훈련 필요성
- 취업 장벽 요인

### Step 3: 생태도 (Eco-map) 작성

클라이언트를 중심으로 환경 체계와의 관계를 시각화합니다.

```
범례: ━━━ 강한 관계 / ─── 보통 관계 / ┄┄┄ 약한 관계 / ╳╳╳ 갈등 관계 / → 자원 흐름 방향

                    [지역사회]
                        ┄┄┄
        [직장/학교] ─── [클라이언트] ━━━ [가족]
                        ╳╳╳               │
                    [이웃/친구]        ━━━ [자녀]

        [종교기관] ┄┄┄               ─── [의료기관]
                    [사회복지기관] ━━━→
```

### Step 4: 욕구 우선순위 매트릭스

| 순위 | 욕구 영역 | 긴급도 | 중요도 | 클라이언트 동의 | 개입 가능성 |
|------|----------|--------|--------|----------------|------------|
| 1 | | 상/중/하 | 상/중/하 | 예/아니오 | 상/중/하 |

- 긴급도와 중요도가 모두 "상"인 욕구를 최우선으로 배치합니다
- **클라이언트 동의** 열이 핵심입니다 — 동의 없는 개입은 계획하지 않습니다

### Step 5: 강점 사정

반드시 문제목록과 동일한 비중으로 강점을 기록합니다:

| 영역 | 강점/자원 |
|------|----------|
| 개인적 강점 | (대처 경험, 동기, 기술 등) |
| 가족 자원 | (지지적 가족, 돌봄 역량 등) |
| 사회적 자원 | (지지체계, 소속 집단 등) |
| 지역사회 자원 | (이용 가능한 서비스, 시설 등) |
| 영적/문화적 자원 | (신앙, 문화적 정체성, 가치관 등) |

### Step 6: 위험요인 스크리닝

| 위험 요인 | 현재 해당 | 수준 | 비고 |
|-----------|----------|------|------|
| 자살/자해 위험 | 예/아니오 | 상/중/하 | |
| 타해 위험 | 예/아니오 | 상/중/하 | |
| 학대/방임 (가해/피해) | 예/아니오 | 상/중/하 | |
| 약물/알코올 남용 | 예/아니오 | 상/중/하 | |
| 가정폭력 | 예/아니오 | 상/중/하 | |
| 주거 상실 위험 | 예/아니오 | 상/중/하 | |

**위험요인 "상" 감지 시**: 즉시 AskUserQuestion으로 실천가에게 알립니다.

## 재사정 (Re-assessment)

기존 사례에 대해 `/assessment`를 다시 실행하면:
1. 이전 사정보고서를 읽어 변화를 비교합니다
2. 척도 점수의 변화를 추적합니다 (예: PHQ-9 15점 → 9점)
3. 새로운 욕구/강점/위험요인을 업데이트합니다
4. `assessment_type: 재사정`으로 기록합니다

## 개인정보 보호 규칙

- 접수기록서와 동일한 개인정보 보호 원칙을 적용합니다
- 척도 원점수는 기록하되, 해석 시 낙인을 피하는 표현을 사용합니다
  - ❌ "심각한 우울증 환자"
  - ✅ "PHQ-9 18점으로 중등도 이상의 우울 증상이 관찰됨"
