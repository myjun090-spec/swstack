# /expense — 지출 관리 스킬

> 역할: 경리
> 영역: 회계/재정

## 개요

지출 증빙을 검토하고 정산서를 생성합니다.
예산 대비 집행률 모니터링, 품의서 작성을 지원합니다.

## 워크플로

### Step 1: 지출 유형 확인

| 유형 | 절차 |
|------|------|
| 일반 지출 | 품의 → 집행 → 증빙 → 정산 |
| 긴급 지출 | 집행 → 사후 품의 → 증빙 → 정산 |
| 카드 지출 | 집행 → 증빙 첨부 → 월말 정산 |

### Step 2: 품의서 작성

사용자 입력을 받아 품의서를 생성합니다:

```markdown
# 지출품의서

- 품의일: {날짜}
- 품의자: {이름}
- 사업명: {해당 사업}
- 예산항목: {관-항-목}

| 품명 | 규격 | 수량 | 단가 | 금액 |
|------|------|------|------|------|
| | | | | |

- 합계: {총액}원
- 지출 사유: {사유}
- 예산 잔액: {잔액}원 (집행 후 잔액: {예상잔액}원)
```

### Step 3: 증빙 체크리스트

| 증빙 유형 | 필수 확인 항목 |
|----------|---------------|
| 세금계산서 | 공급자 사업자번호, 품목, 금액 일치 |
| 카드매출전표 | 가맹점명, 일시, 금액 |
| 간이영수증 | 3만원 이하만 가능, 기관명 기재 |
| 이체 확인서 | 수취인, 금액, 일시 |
| 강사비 지급 | 원천징수 3.3%, 강사 서명, 신분증 사본 |

### Step 4: 정산서 생성

월별/분기별 정산서를 생성합니다.

## 산출물

`.swstack/budgets/expenses/` 디렉토리에 저장합니다.

## 연결 스킬

- `/budget` — 예산 대비 확인
- `/audit-prep` — 증빙 누락 점검
