# /case-conference — 사례회의 스킬

> 역할: 사례회의 진행자
> 유형: 사례관리 보조 (필요 시 실행)

## 개요

복잡 사례나 윤리적 딜레마가 있는 사례에 대해 다학제 팀 사례회의를 구조화합니다.
안건 구성, 회의 진행 가이드, 회의록 작성, 결정사항 추적을 수행합니다.

## 실행 전 확인

1. 해당 사례의 기존 기록(intake, assessment, plan, notes)을 읽습니다
2. 이전 사례회의록이 있으면 참조합니다

## 워크플로

### Step 1: 회의 유형 결정

| 유형 | 목적 | 참석자 |
|------|------|--------|
| 신규사례 배정 | 접수 후 담당자 배정, 초기 방향 논의 | 팀장, 사례관리자 |
| 정기 사례점검 | 진행 중인 사례 점검 | 팀 전체 |
| 위기사례 긴급회의 | 위기 상황 대응 방안 논의 | 팀장, 담당자, 관련 전문가 |
| 종결 검토 | 종결 적절성 판단 | 팀장, 담당자, 슈퍼바이저 |
| 윤리 검토 | 윤리적 딜레마 논의 | 팀 전체, 외부 자문 |

### Step 2: 안건 구성

사용자에게 다음을 질문하여 안건을 구성합니다:

1. 회의 상정 사유
2. 현재까지의 핵심 쟁점
3. 회의에서 결정이 필요한 사항
4. 참석자 목록

### Step 3: 사례 요약 자료 생성

기존 기록을 기반으로 회의용 요약자료를 생성합니다:
- 사례 개요 (1페이지 이내)
- 주요 쟁점
- 실천가 의견
- 논의 요청 사항

### Step 4: 회의록 작성

## 산출물 형식

`.swstack/cases/{case-id}/conferences/{YYYY-MM-DD}-conference.md`에 저장합니다.

```markdown
---
case_id: {case-id}
conference_date: {날짜}
conference_type: {유형}
attendees: [{참석자 목록}]
---

# 사례회의록

## 1. 회의 정보
- 일시: {날짜 시간}
- 유형: {회의 유형}
- 참석자: {이름/직위}
- 안건: {안건 제목}

## 2. 사례 요약
{회의용 사례 요약}

## 3. 논의 내용
{참석자별 의견 및 논의 과정}

## 4. 결정 사항
| 번호 | 결정 내용 | 담당자 | 기한 | 상태 |
|------|----------|--------|------|------|
| 1 | | | | 대기/진행중/완료 |

## 5. 후속 조치
{다음 단계 및 일정}
```

## 연결 스킬

- `/plan-case` — 회의 결정에 따른 계획 수정
- `/supervise` — 슈퍼비전 연계
- `/intervene` — 결정사항 반영한 개입 기록
